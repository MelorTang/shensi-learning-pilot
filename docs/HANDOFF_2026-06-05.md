# Shensi Handoff - 2026-06-05

## What was done

Implemented a **Shensi-side Feishu direct router** that bypasses Hermes LLM
for keyword-triggered actions.

### New files

- `app/feishu/router_helpers.py` — pure helpers: `classify_intent()` and `index_image_path()`
- `tests/test_router.py` — unit tests for intent classifier and index path
- `deploy/systemd/shensi-router.service` — cloud systemd template

### Modified files

- `scripts/run_feishu_ws.py` — added `--router` mode (mutually exclusive with `--card-actions-only`)
- `README.md` — router documented as recommended Feishu path

### What the router does

| Trigger | Latency | Action |
|---------|---------|--------|
| Image | <1s | Download to `~/.hermes/image_cache`, write index, reply "已收到图片...", spawn `shensi-feishu-analysis-latest` in background |
| 慎思分析 | <1s | Resolve indexed/cached image, spawn analysis, reply "正在分析..." |
| 确认入库 | <1s | `POST /hermes/pending/latest/confirm` → reply with result |
| 丢弃 | <1s | `POST /hermes/pending/latest/discard` → reply with result |
| 帮助 | <1s | Reply help text |
| Other text | <1s | Silently ignored |

Key design decisions:
- No LLM ever called — pure keyword matching
- Background analysis spawns via `subprocess.Popen(start_new_session=True)` — fully detached
- Same image cache/index paths as Hermes (`~/.hermes/image_cache`, `~/.hermes/shensi_image_index`)
- Card action callbacks still forwarded to Shensi (same as `--card-actions-only`)

### What was NOT changed

- Database schema — untouched
- Confirm async logic (`_confirm_mistake_core` / `_write_confirmation_artifacts`) — untouched
- Pending modification flow — not restored (per product decision)
- Hermes docs — not deleted
- No OpenClaw, no Gemini direct call

### Cloud deployment

```bash
cd /home/admin/apps/shensi-learning-pilot
git pull
source .venv/bin/activate
python -m pytest

# Install router service
sudo cp deploy/systemd/shensi-router.service /etc/systemd/system/
# or as user:
cp deploy/systemd/shensi-router.service ~/.config/systemd/user/

# Start router
sudo systemctl daemon-reload && sudo systemctl enable shensi-router && sudo systemctl start shensi-router
# or user:
systemctl --user daemon-reload && systemctl --user enable shensi-router && systemctl --user start shensi-router

# Disable Hermes im.message.receive_v1 subscription to avoid duplicates
# Hermes gateway can stay running for free-form conversation via separate bot
# The existing shensi-feishu-card.service is no longer needed when router is active:
#   systemctl --user stop shensi-feishu-card
#   systemctl --user disable shensi-feishu-card

curl http://127.0.0.1:8000/health
```

### Verifying

```bash
systemctl --user status shensi-router --no-pager | head -5
journalctl --user -u shensi-router --no-pager | tail -20
```
