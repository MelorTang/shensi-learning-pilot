## 1. Project goal

Build a runnable MVP for `慎思｜AI 学习诊断与复盘系统` with this closed loop:

Feishu parent uploads homework image
-> Hermes receives command
-> Antigravity/Gemini extracts structured question JSON
-> Shensi ingests analysis
-> deterministic math verification runs when possible
-> parent confirms or discards
-> SQLite + Obsidian write
-> D+1 / D+3 / D+7 review tasks
-> daily / weekly reports
-> Hermes query APIs

## 3. Important architecture decisions

### 3.1 Role split

- Hermes should be a thin Feishu router only.
- Hermes should not directly write SQLite or Obsidian.
- Antigravity/Gemini handles image understanding and structured extraction.
- Shensi is the system of record for:
  - deduplication
  - deterministic verification
  - parent confirmation state
  - SQLite writes
  - Obsidian writes
  - review tasks
  - reports

### 3.2 Preferred cloud flow

Feishu
-> Hermes Gateway
-> `/home/admin/bin/shensi-feishu-analysis-latest`
-> `/home/admin/bin/shensi-antigravity-submit`
-> `/home/admin/bin/shensi-antigravity-vision`
-> `POST http://127.0.0.1:8000/ingest/mistake-analysis`
-> `POST http://127.0.0.1:8000/hermes/pending/latest/card/send`
-> Feishu interactive card

### 3.3 Important product behavior

- Hermes should not use its own multimodal answer as the final homework judgment.
- If a question is not covered by deterministic verification, the parent-facing output must say it is only model-judged / needs review.
- If a Feishu interactive card is sent successfully, Hermes should not send an extra long text report.

### 5.2 Critical env requirement

This was a real production issue and is now documented:

`shensi.service` must load both:

- `/home/admin/apps/shensi-learning-pilot/.env`
- `/home/admin/.hermes/.env`

Without that, analysis can finish and enter `waiting_confirmation`, but
`/hermes/pending/latest/card/send` fails with `HTTP 502`, because the FastAPI
process cannot read Feishu credentials.

Expected `shensi.service` fragment:

```ini
EnvironmentFile=/home/admin/apps/shensi-learning-pilot/.env
EnvironmentFile=-/home/admin/.hermes/.env
```

`shensi-feishu-card.service` should also load both files.

### 5.3 Critical finding from debugging

The earlier symptom:

- Feishu shows: `正在分析这张错题，完成后我会发确认卡片。`
- Then nothing happens

Root cause found:

- background analysis succeeded
- Shensi pending item existed
- final Feishu card send failed with `HTTP 502`
- reason: `shensi.service` lacked Feishu env from `~/.hermes/.env`

This is fixed operationally on cloud already.

## 9. Current known issues

### 9.1 Major functional issue still open

Deterministic verification coverage is still too shallow for grade 8 / grade 9
composite math questions.

Current behavior is acceptable for MVP because:

- unsupported items remain pending parent review
- cards now clearly distinguish verified vs model-only judgments

But accuracy still depends heavily on vision/model output for many advanced items.

### 9.2 Hermes image selection is still risky

The cloud script `shensi-feishu-analysis-latest` currently finds the newest file
in the global Hermes image cache directory.

This is a known shortcut and can pick the wrong image if multiple chats/images
exist close together.

Desired future behavior:

- pick latest image from the same chat
- ideally same sender too
- do not rely on global newest cache file

Small improvement already landed after the first handoff draft:

- `shensi-feishu-analysis-latest` now accepts an optional explicit image path as
  argument 5
- if Hermes can resolve the exact current image file, it should pass that path
  instead of relying on the global-cache fallback

Additional follow-up implemented later:

- `scripts/cloud/shensi-index-image` indexes `chat_id + sender_id -> image_path`
- `shensi-feishu-analysis-latest` reads that index before falling back to the
  global newest cache image
- wrapper timing logs now include `resolve_image`, `submit`, `card_send`, and
  total elapsed timings

### 9.3 Hermes prompt / routing may still drift

Even with the skill file added, the next agent should confirm:

- live Hermes session is actually loading the skill
- Hermes is not silently using its own multimodal explanation path before Shensi
- Hermes is not leaking internal tool chatter into Feishu

### 9.4 Some Chinese text still shows mojibake in shell logs

This is mainly terminal / locale rendering noise and not necessarily a data
corruption problem inside the app. API JSON and files were still usable.
